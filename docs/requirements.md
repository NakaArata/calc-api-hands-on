# 機能要件（requirements）

## 1. 目的
- Azure Functions 上で動作する、簡単な計算（掛け算・割り算）を提供する HTTP API を提供する。
- ブラウザで URL を開いた際に、レスポンス本文として結果がそのまま表示されることを目的とする。

## 2. 対象範囲
- 対象 API: 掛け算 API / 割り算 API の 2 つ
- 通信: HTTP GET のみ
- 入力: クエリストリング `A` と `B`
- 出力: `text/plain`

## 3. API 一覧
| API | HTTP | パス | 概要 |
|---|---|---|---|
| Multiply | GET | `/mul` | `A * B` を返す |
| Divide | GET | `/div` | `A / B` を返す |

## 4. 共通仕様
### 4.1. リクエスト
- クエリパラメータ
  - `A`: 数値（小数可）
  - `B`: 数値（小数可）

### 4.2. レスポンス
- 成功時
  - HTTP ステータス: `200 OK`
  - `Content-Type`: `text/plain; charset=utf-8`
  - 本文: 計算結果の文字列表現（例: `6` / `2.5`）

### 4.3. バリデーション・エラー
以下の場合は共通でエラーとして扱う。
- `A` または `B` が未指定
- `A` または `B` が数値として解釈できない

- エラー時
  - HTTP ステータス: `400 Bad Request`
  - `Content-Type`: `text/plain; charset=utf-8`
  - 本文: エラーメッセージ（人が読める1行のテキスト）

### 4.4. 割り算の追加ルール
- `B = 0`（ゼロ除算）の場合
  - HTTP ステータス: `400 Bad Request`
  - 本文例: `B must not be 0`（文言は実装で多少変わってよい）

## 5. 例
### 5.1. 掛け算
- リクエスト例: `/mul?A=2&B=3`
- レスポンス本文例: `6`

### 5.2. 割り算
- リクエスト例: `/div?A=5&B=2`
- レスポンス本文例: `2.5`

### 5.3. エラー（未指定）
- リクエスト例: `/mul?A=2`
- ステータス: `400`
- 本文例: `A and B are required`

### 5.4. エラー（ゼロ除算）
- リクエスト例: `/div?A=1&B=0`
- ステータス: `400`
- 本文例: `B must not be 0`

## 6. 前提・補足
- 数値の扱いは「小数可」を前提とする（実装では Python の `float` 等で処理）。
- 結果の丸め（桁数固定など）は行わない（必要になった時点で要件追加）。
